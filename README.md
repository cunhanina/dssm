An academic repository tracking the progression of the Software Design for Mechatronic Systems course at UniSENAI PR. It focuses on building high-performance, structured algorithms utilizing C and C++.

**The Problem:** Modern mechatronic engineering requires highly efficient, robust software to control complex industrial hardware and processes. Abstract programming concepts often fail to address the memory constraints and architectural needs of physical systems.

**The Solution:** A progressive workspace that bridges raw logic with engineering applications. This repository transitions from foundational algorithms to advanced modularization, culminating in software architectures specifically designed for embedded mechatronic systems.

---

## 📸 Capabilities
* **Algorithmic Foundations:** Implementations of basic structures, operators, conditionals, and repetition loops to solve mathematical logic problems.
* **Advanced Data Handling:** Manipulation of vectors, matrices, character chains, and file I/O operations for reading and writing persistent data.
* **Modular Architecture:** Deep dive into functions, scope management, parameter passing by reference, and the creation of custom libraries for project organization.
* **Practical Projects (APS):** Contains functional academic challenges, including a mathematical calculator (APS 1) and a documented Fibonacci sequence generator in C++ (APS 2).
* **Embedded Focus:** Architectural principles, build processes, and structural patterns aimed at robustness in embedded software environments.

| Component | Technology | Description |

| **Debugging** | GDB | Tooling for diagnosing memory and logic errors |

## 🧠 System Architecture
This repository is structured around the chronological progression of the course syllabus:
1.  **Core Logic (Weeks 1-5):** Focuses on basic syntax, mathematical libraries, and control structures.
2.  **Data & Modularity (Weeks 6-13):** Introduces complex data types (matrices, structures), pass-by-reference mechanics, and splitting code into custom header/source libraries.
3.  **Deployment (Weeks 14-20):** Covers the actual compilation process, build mechanics, and best practices for structuring robust software for embedded applications.
